ScholarViewは、AT Protocolを基盤とした**分散型学術査読・熟議プラットフォーム**です。研究者が自身のデータを所有し、オープンな環境で査読と議論を行える次世代の学術インフラを提供します。

## ScholarViewが解決する課題

現代の学術出版システムは、中央集権的な出版社によるデータの囲い込み、不透明な査読プロセス、そして高額な購読料といった課題を抱えています。ScholarViewは、**DeSci (Decentralized Science)** の原則に基づき、これらの課題を技術で解決します。

- **データの主権**: 論文データは研究者自身のPDS（Personal Data Server）に保存され、特定のプラットフォームに依存しません。
- **透明な査読プロセス**: Blueskyのソーシャル基盤を活用し、誰でも閲覧・参加可能なオープンな熟議を実現します。
- **ローカルファースト**: ユーザー自身がデータをインデックスし、手元で管理することで、検閲耐性と高いプライバシーを確保します。

## コア・コンセプト

### 1. AT Protocolによる構造化データ管理
論文は単なるPDFではなく、AT Protocol上の構造化されたレコード（`sci.peer.article`）として管理されます。これにより、論文の一部を引用したり、プログラムから解析したりすることが容易になります。

### 2. Blueskyとのシームレスな統合
査読コメントや議論はすべてBlueskyのポスト（`app.bsky.feed.post`）として扱われます。巨大なソーシャルネットワークの力を借りることで、研究内容がより広く拡散され、多様なフィードバックを得ることが可能になります。

### 3. ハイブリッド・ブロック編集
TeXとMarkdownの利点を組み合わせた独自のブロックエディタにより、数式を含む高度な学術文章を直感的に執筆できます。

## ScholarViewの構成

ScholarViewは、以下の3つの主要なコンポーネントで構成されています。

1.  **Workspace**: 論文の執筆、ファイル管理、プレビューを行うデスクトップ級のWeb UI。
2.  **Lexicon**: 分散型ネットワーク上でのデータ形式を定義するスキーマ群。
3.  **AppView (Local)**: ネットワーク上のデータを収集し、自分専用のデータベースを構築する同期エンジン。
