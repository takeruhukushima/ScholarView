# ScholarView 要件定義書 (機能網羅版)

## 1. プロジェクト概要
ScholarViewは、AT Protocolを活用した非中央集権的な科学（DeSci）のためのプラットフォームです。研究者が論文を自律的に公開し、BlueSkyのエコシステムを通じて透明性の高い議論を行うことを可能にします。

## 2. システム構成
- **フロントエンド**: Next.js (React 19), Tailwind CSS
- **通信プロトコル**: AT Protocol (Lexicon: `sci.peer.article`)
- **認証**: AT Protocol OAuth
- **データストア**: ブラウザローカルストア（IndexedDB等） + AT Protocol Repository

## 3. 機能要件 (MECE分類)

### 3.1 記事・公開管理 (Remote Article Management)
AT Protocol リポジトリ上のレコード操作。
- **新規公開 (Create)**:
    - `sci.peer.article` レコードの作成。
    - タイトル（最大300字）、著者（DID/所属含む）、本文ブロック、文献目録、画像アセットの永続化。
- **更新 (Update/Put)**:
    - 既存レコードの置換。
    - 更新履歴の整合性維持（既存の `createdAt` の保持）。
- **削除 (Delete)**:
    - リポジトリからのレコード削除。
    - 関連する告知投稿（Announcement）の自動削除（カスケード削除）。
- **取得 (Fetch/Resolve)**:
    - ローカルキャッシュ -> パブリックリレー (bsky.app) -> 直接PDS (PLC解決経由) の順でフォールバック取得。

### 3.2 ワークスペース・ドラフト管理 (Local Content Management)
ブラウザ上の執筆環境およびローカル永続化。
- **ファイルシステム**:
    - フォルダ・ファイル（.md, .tex, .bib, 画像）の作成・削除・リネーム。
    - ドラッグ＆ドロップによる移動およびソート順の管理。
- **ドラフト機能**:
    - 未公開コンテンツの一時保存（最大60,000文字）。
    - 公開前プレビューおよび編集。
- **インポート解決 (Resolve Imports)**:
    - 文書内の `workspace://` 参照を実際のパスやコンテンツに解決。
    - 画像パスの自動補正。
- **初期セットアップ (Seeding)**:
    - 新規ユーザー向けのチュートリアル用フォルダ・ファイルの自動生成。

### 3.3 ソーシャル・ディスカッション (Social & Interaction)
BlueSky (app.bsky.feed.post) との連携。
- **告知 (Broadcast)**:
    - 公開/更新時にBlueSkyへ投稿。
    - 投稿テキストのカスタマイズと記事URL（カード形式）の埋め込み。
    - 更新時は既存スレッドの末尾（Thread Tail）への返信として投稿。
- **インラインコメント**:
    - 記事の特定箇所を引用（最大280字）したBlueSky投稿。
- **ディスカッション統合**:
    - BlueSky上のライブスレッドとローカルのインラインコメントをマージして表示。
    - スレッドのルート投稿（Announcement）の自動探索・紐付け。
- **エンゲージメント**:
    - 投稿に対する「いいね (Like)」「リポスト (Repost)」「返信 (Reply)」の実行と同期。

### 3.4 同期・リソース管理 (Sync & Resource Handling)
リモートとローカルの橋渡し。
- **リポジトリ同期 (Sync articles)**:
    - リポジトリ上の全記事をスキャンし、未リンクの記事をワークスペースファイルとして自動生成。
    - リンク済みファイルの最新化。
- **画像アセット処理**:
    - 文書内のローカル画像参照を検出し、公開時にAT ProtocolのBlobとしてアップロード。
    - Blobリンクを記事レコードに埋め込み。
- **文献管理 (Citations)**:
    - `.bib` ファイルの解析。
    - 文献目録の正規化、重複排除、シリアライズ。

## 4. 非機能要件
- **セキュリティ**:
    - OAuthセッションに基づくDID検証。
    - 自身の所有するレコード以外への書き込み制限。
- **信頼性**:
    - ネットワークエラー時のリトライおよびエラーハンドリング（HttpErrorクラス）。
    - PDSエンドポイントの動的解決。
- **パフォーマンス**:
    - 記事取得時のページネーション（100件単位、最大20ページ）。
    - 同期処理の並行実行制御（In-flight管理）。
